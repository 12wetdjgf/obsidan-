## 🧱 先搞懂基础设施！

在写代码之前，你得先认识几个"朋友"：

### 坐标系统
```
     col 0  col 1  col 2  col 3
row 3  |    |    |    |    |    ← 最上面是 y=3
row 2  |    |    |    |    |
row 1  |    |    |    |    |
row 0  |    |    |    |    |    ← 最下面是 y=0
```

**注意！！！** (0,0) 在**左下角**，不是左上角！这个超容易搞混的！😤

### 你要用到的 API

| 类 | 方法 | 作用 |

| `Board` | `board.size()` | 返回棋盘大小（通常是4） |

| `Board` | `board.setViewingPerspective(Side s)` | 改变视角（Task 9 用） |
| `Tile` | `t.value()` | 返回这个格子的数值 |

## 🟢 Task 1: emptySpaceExists()

### 题意
> 棋盘上有没有空格子？有就返回 `true`。

### 思路引导

这个超简单的啦～就是个**双重循环**遍历整个棋盘：

```
对于每一列 x（从 0 到 size-1）:
    对于每一行 y（从 0 到 size-1）:
        拿到 tile(x, y)
        如果是 null → 说明有空格！返回 true
全部看完了都没有 null → 返回 false
```

### 💡 关键提示
- `board.tile(x, y)` 返回 `null` 就代表这个位置是空的
- 别忘了遍历**所有**位置
- 用 `board.size()` 获取大小，别硬编码 `4`

### 测试方法
运行 `TestEmptySpace.java`，8 个测试全过就 OK！

## 🟢 Task 2: maxTileExists()

### 题意
> 棋盘上有没有值为 2048 的格子？

### 思路引导

跟 Task 1 几乎一样！双重循环遍历：

```
对于每个位置 (x, y):
    拿到 tile
    如果 tile 不是 null 而且 tile.value() == MAX_PIECE:
        返回 true
全部看完 → 返回 false
```

### 💡 关键提示
- 用 `MAX_PIECE` 这个变量，**不要**写 `2048`！（这叫避免"魔法数字"）
- 一定要**先检查 null**，再调用 `.value()`！不然会 `NullPointerException`！

```java
// ❌ 错误：如果 tile 是 null，调用 .value() 会炸！
if (tile.value() == MAX_PIECE) ...

// ✅ 正确：先判断不为 null
if (tile != null && tile.value() == MAX_PIECE) ...
```

这个坑我当时也踩过，超经典的 😂

## 🟢 Task 3: atLeastOneMoveExists()

### 题意
> 玩家还能不能动？（还有没有合法操作）

### 什么时候能动？
1. 棋盘上**有空格**（嘿，你刚写了 `emptySpaceExists()`！直接调用！）
2. 或者**有相邻的格子值相同**（可以合并）

### 思路引导

```
如果 emptySpaceExists() 返回 true → 直接返回 true（复用你写的代码！）

否则，检查所有相邻的格子对：
    对于每个位置 (x, y):
        检查它右边的邻居 (x+1, y)：值一样吗？
        检查它上面的邻居 (x, y+1)：值一样吗？
        （只需要检查右边和上面，不需要四个方向，想想为什么？）
    如果找到任何一对相同的 → 返回 true

全部看完没有 → 返回 false
```

### 💡 关键提示
- **为什么只检查右边和上面？** 因为如果 A 在 B 左边且相同，那从 B 的角度看，A 就在它左边——但从 A 的角度看 B 在它右边，所以我们遍历到 A 时就已经发现了！避免重复检查～
- **注意边界！** 最右列没有右邻居，最上行没有上邻居，别数组越界了！

## 🟡 Task 4: 理解滑动规则（不用写代码）

这是**阅读理解**环节！三条核心规则：

### 规则 1：相同值合并
两个相同的值碰到一起 → 合并成两倍

### 规则 2：合并后不再合并
`[X, 2, 2, 4]` 向左滑 → `[4, 4, X, X]` ✅
**不是** `[8, X, X, X]` ❌（那个 4 是刚合出来的，不能再合了！）

### 规则 3：同方向三个一样，前两个合并
`[X, 2, 2, 2]` 向左滑 → `[4, 2, X, X]` ✅
**不是** `[2, 4, X, X]` ❌（靠近滑动方向的两个先合并！）

### 推论：四个一样 → 两两合并
`[4, 4, 4, 4]` 向左滑 → `[8, 8, X, X]`

## 🟡 Task 5: moveTileUpAsFarAsPossible(x, y)

### 题意
> 把 (x, y) 位置的格子**尽量往上移**（先不考虑合并）

### 整体架构

项目帮你设计了一个**自底向上**的拆解方式，超级优雅：

```
移动单个格子（Task 5-6）
    ↓ 被调用
倾斜一列（Task 7）
    ↓ 被调用  
倾斜整个棋盘向上（Task 8）
    ↓ 被调用
倾斜四个方向（Task 9）
```

### 思路引导

```
拿到 (x, y) 位置的 tile
如果是 null → 啥也不做，直接返回

从 y 的上一行开始，一直往上扫描：
    如果遇到空格 → 继续往上
    如果遇到有格子 → 停下来（这个格子挡住了）
    
把 tile move 到能到达的最高位置
```

### 💡 关键提示
- `board.move(x, y, tile)` **只能调用一次**！不能一格一格挪！
- 如果 tile 本来就不需要动 → **不要调用 move**！（Task 10 的一个大坑）
- 你需要找到"最终目的地"是哪个 y 坐标

**想象自己是那个格子**：你往上看，一路都是空的你就继续飞，碰到别的格子或者到顶了就停下来。最终你落脚的位置就是你要 move 到的地方。

## 🟡 Task 6: 加入合并逻辑

### 修改 moveTileUpAsFarAsPossible

在 Task 5 的基础上，加一个判断：

```
...往上扫描的时候...
如果遇到有格子：
    如果那个格子的值跟我一样 AND 那个格子没被合并过：
        → 合并！（move 到那个格子的位置）
    否则：
        → 停在那个格子下面一格
```

### 💡 关键提示
- 用 `tile.wasMerged()` 检查目标格子是否已经在这一轮合并过
- `board.move()` 会自动处理合并的值更新
- 合并 = 移到那个同值格子的位置上（不是它下面！）

## 🟡 Task 7: tiltColumn(int x)

### 题意
> 把第 x 列所有格子都往上滑

### 思路引导

你已经有了 `moveTileUpAsFarAsPossible`，现在只要对这一列的每个格子调用它就行了！

**但是！顺序超重要！** 🚨

```
从上往下处理？还是从下往上处理？
```

想想看：如果你从最下面的格子开始处理，它往上飞的路上可能撞到还没处理过的格子...那画面太美我不敢看 😂

**答案：从上往下处理！**（从 y = size-1 到 y = 0）
- 最上面的格子先动（或者不动）
- 然后是下面的格子，它往上飞的时候，上面的格子已经就位了

但等等，最上面那行（y = size-1）的格子需要处理吗？它已经在顶上了...🤔 你自己想想要不要跳过它～

## 🟡 Task 9: 四个方向的滑动

### 核心思路

这里有个**超级聪明的技巧**！不需要写四份代码！

想象你在玩手机游戏，想往左滑——你可以把手机顺时针转90度，然后往上滑，效果一样！

`setViewingPerspective` 就是干这个的！

### 伪代码

```java
board.setViewingPerspective(side);  // 假装 side 方向是"上"

// 然后用你已经写好的"往上滑"的逻辑！
// （对每一列调用 tiltColumn）

board.setViewingPerspective(Side.NORTH);  // 别忘了转回来！！！
```

### 💡 关键提示
- **一定要在最后设回 `Side.NORTH`**！不然后面全乱套！
- 就加两行代码，你之前 Task 8 写的逻辑一个字都不用改！

### 规则

### 思路引导

## 🐛 常见Bug速查表

| 症状 | 可能原因 |
|------|---------|
| `NullPointerException` | 对 null 的 tile 调用了 `.value()` 或 `.move()` |
| `ArrayIndexOutOfBoundsException` | 循环边界写错了，x 或 y 超出范围 |
| 格子值莫名翻倍 | 对没有移动的格子调用了 `move`（移到原位=自我合并！） |
| 合并两次 | 没检查 `wasMerged()` |
| 只有向上能动 | 忘了 `setViewingPerspective` |
| 方向全乱了 | 忘了在最后设回 `Side.NORTH` |
