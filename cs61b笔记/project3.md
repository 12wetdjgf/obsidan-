## 🧩 阶段一：3A 世界生成（最核心！）

这是整个项目的**地基**，你要写一个算法，能根据一个**种子（seed）**随机生成一个2D瓦片世界。

### 世界必须满足的硬性要求：

```
✅ 必须有明显区分的「房间」和「走廊」
✅ 至少部分房间是矩形的
✅ 走廊必须有拐弯的（不能全是直线）
✅ 不能有死胡同走廊
✅ 墙壁、地板、空白区域视觉上要能区分开
✅ 房间和走廊之间不能有缝隙
✅ 所有房间都必须可达（不能有孤岛房间）
✅ 房间不能超出世界边界
✅ 世界利用率 > 50%（不能大片空白）
```

### 随机性要求：

```
✅ 房间数量随机
✅ 房间位置随机
✅ 房间大小随机
✅ 走廊长度随机（宽度固定为1格）
✅ 不同种子生成的世界要明显不同
```

### 💡 我的思路建议（不是给你答案哦！是教你怎么想）

好，这里是最关键的部分了！世界生成算法有很多种方式，我给你几个**思考方向**：

#### 方法一：先放房间，再连走廊（推荐新手）

```
伪代码思路：
1. 创建一个空的 TETile[][] 数组，全部填充为 Tileset.NOTHING
2. 随机生成 N 个房间（N是随机的）
   - 每个房间有随机的 (x, y) 位置、随机的宽和高
   - 检查房间是否越界、是否和已有房间重叠
   - 如果合法，把房间"画"到数组上（地板 + 墙壁）
3. 把所有房间连起来
   - 可以按照房间中心点排序
   - 相邻房间之间用 L 形走廊连接
4. 最后给所有地板周围加上墙壁
```

这个方法的核心思想是：

```
房间 = 一块矩形地板 + 四周的墙
走廊 = 宽度为1的地板通道 + 两侧的墙
```

#### 方法二：BSP（二叉空间分割）

```
1. 把整个世界看作一个大矩形
2. 随机选择水平或垂直切割，分成两半
3. 递归地继续切割，直到子区域足够小
4. 在每个最小区域内随机放一个房间
5. 自底向上，把相邻的子区域用走廊连起来
```

这个方法生成的世界会比较**均匀分布**，不容易出现大片空白。

> 🎯 **重点提示**：spec 说了"你大概率会扔掉第一版算法"，所以不要怕写错！先写一个能跑的，再慢慢优化。

## 🎮 阶段二：3B 交互性

世界生成搞定后，接下来让玩家能**走进去玩**！

### 核心功能清单：

#### 1. 角色移动
```
W = 上    A = 左    S = 下    D = 右
- 撞墙不能移动（也不能报错）
- 必须是确定性的！同样的种子 + 同样的按键 = 同样的结果
```

#### 2. HUD（抬头显示）
```
- 鼠标悬停在某个瓦片上时，显示该瓦片的描述
- 比如鼠标在墙上就显示 "wall"
- ⚠️ 不能闪烁！
```

#### 3. 存档和读档（这个是难点哦！）

```
:Q = 保存并退出（注意是冒号+Q，不区分大小写）

保存的状态必须包括：
- 世界的所有瓦片
- 角色位置
- Random 对象的状态！（这个容易忘）
- 其他任何会影响游戏状态的东西
```

**存档原理**：用文件持久化！参考 Lab 9 的 `FileUtils`。

```
思路：
- 保存时：把所有游戏状态序列化/写入 .txt 文件
- 读取时：从文件中读取并恢复状态
- 文件只能用 .txt 后缀！
```

#### 4. `getWorldFromInput` 要支持移动和存档！

这是最容易被绊倒的地方，看好了：

```
"N543SWWWWAA"
 │  ││    ││
 │  ││    └┘ 左移2次
 │  │└────── 移动: 上移4次  
 │  └─────── S: 种子输入结束
 │            
 └──── N + 543: 新世界，种子543
```

而且这些调用必须产生**完全一样**的结果：

```java
// 以下四种方式，最终世界状态必须完全相同！
getWorldFromInput("N999SDDDWWWDDD")

getWorldFromInput("N999SDDD:Q")  → getWorldFromInput("LWWWDDD")

getWorldFromInput("N999SDDD:Q")  → getWorldFromInput("LWWW:Q")  → getWorldFromInput("LDDD")

getWorldFromInput("N999SDDD:Q")  → getWorldFromInput("L:Q")  
    → getWorldFromInput("L:Q")  → getWorldFromInput("LWWWDDD")
```

这意味着你的**存档和读档必须完美保存所有状态**！
