## 🔧 先学会这几个核心技能

### 1. 断点（Breakpoint）是什么？

想象你在看一部电影，你按了暂停键 ⏸️ —— 断点就是这个暂停键！

**怎么设？** 在 IntelliJ 里，点击代码行号**右边的空白区域**，会出现一个 🔴 红色圆点，这就是断点。

**效果：** 程序运行到这一行时会**暂停在执行这行之前**，然后你就可以偷看所有变量的值了！

### 2. 怎么启动 Debugger？

不是点那个普通的绿色 ▶️ 运行按钮哦！而是点旁边那个 🪲 **小虫子图标**（Debug 按钮）。

### 3. Debugger 窗口里的按钮们

| 按钮 | 功能 | 我的理解 |
|------|------|---------|
| ⏹ Stop | 停止调试 | 不玩了，收工 |
| ▶️ Resume | 继续运行到下一个断点 | 快进到下一个暂停点 |
| ⏭ Step Over | 执行当前行，跳到下一行 | 一步一步走，但不进入方法内部 |
| ⬇️ Step Into | 进入当前行调用的方法里面 | 钻进去看细节 |
| ⬆️ Step Out | 跳出当前方法 | 看够了，出来 |

## 💣 Phase 0：找字符串密码

### 它在干什么？

```
phase0 方法内部：
1. 生成一个秘密字符串 correctPassword（通过 shufflePassword 方法，你不需要看懂它）
2. 把你传入的 password 和 correctPassword 做比较
3. 匹配就通过，不匹配就 BOOM 💥
```

### 🧠 怎么解？思路引导

1. 打开 `Bomb.java`，在 `phase0` 方法里找到**比较密码的那一行**，设一个断点
2. 从 `BombTest.java` 的 `testBombPhase0` 启动 **Debug 模式**
3. 程序暂停后，看右边的变量面板 👀
4. 找到 `correctPassword` 这个变量，它的值就是答案！
5. 把这个值填到 `BombMain.java` 里 `phase0` 的参数中

> 💡 **小提示：** 断点设在比较语句那里最有效，因为那时候 `correctPassword` 已经生成好了！

## 💣 Phase 1：找 IntList 密码

### 它在干什么？

跟 Phase 0 类似，但这次密码不是 String，而是一个 **IntList**（链表）！

```
phase1 方法内部：
1. 生成一个秘密的 IntList 叫 correctIntListPassword
2. 把你传入的 IntList 和它比较
3. 匹配就通过
```

### 🧠 怎么解？思路引导

1. 在 `phase1` 方法里设断点
2. Debug 启动 `testBombPhase1`
3. 这次重点来了 —— **用 Java Visualizer！**

   > 因为 IntList 是链表结构，在变量面板里你得一层一层点开 dropdown，超烦的。Java Visualizer 会画出**箱子和指针的图**，一目了然！

4. 在 debugger 窗口底部找到 **"Java Visualizer"** 这个 tab，点开它
5. 你会看到类似这样的结构：

   ```
   [3] -> [1] -> [4] -> [1] -> null
   ```
   
   （数字是我瞎编的哈，你要自己去看真实值）

6. 然后在 `BombMain.java` 里用 `IntList.of(...)` 来构造答案：

   ```java
   // 假如你看到的是 3 -> 1 -> 4 -> 1
   IntList.of(3, 1, 4, 1)
   ```

> 💡 **小提示：** `IntList.of()` 这个方法超方便！比手写 `new IntList(1, new IntList(2, ...))` 优雅一万倍～

## 💣 Phase 2：条件断点登场！

### 它在干什么？

这个是最酷的一关！🔥

```
phase2 方法内部：
1. 往一个 Set 里塞了 100,000 个随机整数
2. 用 for-each 循环遍历这个 Set，同时用变量 i 计数
3. 当 i == 1337（第1338次迭代）时，检查你的 password 是否等于当前的 number
```

### 问题是什么？

你不能手动 step over 十万次对吧？那得点到手断 😂

### 🧠 怎么解？条件断点！

这就是**条件断点（Conditional Breakpoint）**的用武之地：

1. 在 `phase2` 的循环体内部设一个断点（在检查密码的那行附近）
2. **右键点击那个红色断点圆点** → 选择 "Edit Breakpoint"
3. 在弹出的条件框里输入：

   ```
   i == 1337
   ```

4. 这样程序只会在第 1338 次迭代时暂停！
5. 暂停后，查看当前 `number` 的值
6. 那个值就是你的密码！填到 `BombMain.java` 里

> 💡 **为什么是 1337？** 因为 Java 是从 0 开始计数的，第 1338 次迭代时 `i` 的值是 1337。（而且 1337 = "leet"，程序员梗 😎）
