## 📚 先搞懂大概念：什么是 Disjoint Sets（并查集）？

想象一下你在学校里，每个人一开始都是**独来独往**的，互相不认识。然后慢慢地，有人交朋友了——

- **小明**和**小红**成了朋友 → 他们现在是一个**朋友圈**
- **小刚**和**小美**成了朋友 → 又一个**朋友圈**
- 然后**小明**和**小刚**认识了 → 💥两个朋友圈**合并**了！

这就是并查集！它只做两件事：

| 操作 | 做什么 | 类比 |
|------|--------|------|
| **`union(a, b)`** | 把 a 和 b 所在的集合合并 | 两个朋友圈合并 |
| **`find(a)`** | 找到 a 属于哪个集合（找到"老大"） | 找到朋友圈的代表人物 |

---

## 🌲 核心思想：用树来表示集合

这是最妙的地方！我们用**树**来表示每个集合，树的**根节点（root）**就是这个集合的"老大"。

### 数据结构就是一个数组！

```
int[] parent;
```

- `parent[i]` 存的是节点 `i` 的**父节点**
- 如果 `i` 是根节点，我们存一个**负数**，表示这棵树的大小（取负值）

比如一开始有5个元素：

```
index:   0    1    2    3    4
value:  -1   -1   -1   -1   -1
```

每个人都是自己的老大，size = 1，所以存 `-1`（就是 -size 嘛）

## 🔧 三个优化层级，一步步变强！

### Level 1: Quick Union（基础版）

就是简单地把一棵树的根挂到另一棵树的根下面。

**问题**：可能会变成一条长链！像这样 👇

```
0 → 1 → 2 → 3 → 4    （超级瘦高，find要跑很久！）
```

最坏情况 `find` 是 O(N)，太慢了！😫

### Level 2: Weighted Quick Union（加权合并）💪

**核心思想**：合并的时候，把**小树挂到大树下面**！

为什么？因为这样树不会长太高！最大深度是 **O(log N)**！

> 直觉理解：你的深度只有在你所在的小树被挂到更大的树下面时才会+1，而每次这样做，你所在的树的大小至少**翻倍**。翻倍最多 log N 次就到 N 了！

**tie-breaking**（这个lab的规定）：如果两棵树一样大，把 **v1 的根挂到 v2 的根** 下面。

### Level 3: Path Compression（路径压缩）⚡

**核心思想**：每次 `find` 的时候，把路上经过的**所有节点**直接连到根节点！

```
之前:  5 → 3 → 1 → 0(root)

调用 find(5) 之后:

    0(root)
   /  |  \
  1   3   5     ← 全都直接连到根了！
```

下次再 `find(5)` 就是 O(1) 了，是不是超聪明！🧠

## 💻 现在来写代码吧！一个方法一个方法来～

### 1️⃣ 构造函数 `UnionFind(int N)`

这个最简单啦～

**你要做的事：**
- 创建一个大小为 N 的数组
- 把每个位置初始化为 `-1`（每个元素自成一个集合，size = 1）

> 💡 提示：想想为什么是 -1？因为我们约定根节点存 **-size**，初始 size = 1

---

### 2️⃣ `int find(int v)` ⭐ 最重要的方法！

**目标**：找到 v 所在集合的根节点，同时做**路径压缩**。

**思路：**

```
find(v):
    1. 如果 v 是无效的 → 抛出 IllegalArgumentException
    2. 先找到根节点（怎么判断是根？parent[v] < 0！）
    3. 再把路径上所有节点的 parent 都指向根
    4. 返回根
```

**给你两种实现思路（选一种）：**

**思路A - 两遍遍历：**
```
第一遍：从 v 一路向上走到 root
第二遍：再从 v 走一遍，把路上每个节点的 parent 改成 root
```

**思路B - 递归：**
```
如果 v 是根，返回 v
否则：parent[v] = find(parent[v])  ← 递归 + 顺便压缩
返回 parent[v]
```

> ⚠️ 别忘了验证输入！v 如果 < 0 或者 >= N，要抛异常哦！

---

### 3️⃣ `int parent(int v)`

超简单！直接返回 `parent[v]` 就好～

> 别忘了检查 v 是否合法！

---

### 4️⃣ `int sizeOf(int v)`

**思路：**
1. 用 `find(v)` 找到 v 的根节点 root
2. 根节点存的是 `-size`，所以返回 `-parent[root]`

```
return -parent[find(v)];
```

就一行！优雅～✨

---

### 5️⃣ `boolean connected(int v1, int v2)`

**两个元素连通 ⟺ 它们的根节点相同！**

```
return find(v1) == find(v2);
```

也是一行！简洁就是美！💅

---

### 6️⃣ `void union(int v1, int v2)` ⭐ 第二重要的方法！

这个稍微复杂一点，但想清楚就不难：

```
union(v1, v2):
    1. 找到 v1 的根 root1 = find(v1)
    2. 找到 v2 的根 root2 = find(v2)
    3. 如果 root1 == root2 → 已经连通了，直接 return
    4. 比较两棵树的大小：
       - 如果 size(root1) > size(root2)：
           把 root2 挂到 root1 下面
           更新 root1 的 size
       - 否则（包括相等的情况！）：
           把 root1 挂到 root2 下面  ← tie-breaking!
           更新 root2 的 size
```

**具体操作（假设把 root1 挂到 root2 下面）：**
```java
parent[root2] += parent[root1];  // 更新大小（两个负数相加）
parent[root1] = root2;           // root1 的父节点变成 root2
```

> 🎯 **重点**：tie-breaking 规则是 size 相等时，v1 的根挂到 v2 的根下面！这不是"小的挂到大的下面"，而是题目规定的顺序，别搞反了！

## 🧪 一个完整的例子帮你理解

```
初始状态：[−1, −1, −1, −1, −1]
           0    1    2    3    4

union(0, 1):  root0=0, root1=1, size相等 → 0挂到1下面
           [1, −2, −1, −1, −1]
           
           1
           |
           0

union(2, 3):  root2=2, root3=3, size相等 → 2挂到3下面  
           [1, −2, 3, −2, −1]
           
           1     3
           |     |
           0     2

union(0, 2):  find(0)=1, find(2)=3, size相等 → 1挂到3下面
           [1, 3, 3, −4, −1]
           
              3
            /   \
           1     2
           |
           0

find(0):  0→1→3(root!)  路径压缩后 0 直接连到 3
           [3, 3, 3, −4, −1]
           
              3
            / | \
           1  2  0    ← 0 直接连到根了！
```

## ⚠️ 容易踩的坑

1. **路径压缩不要改 size！** 路径压缩只改 parent 指向，不改根的 size 值
2. **validate 输入！** 每个 public 方法都要检查参数是否合法
3. **union 时别忘了处理已经连通的情况**（find 返回同一个根就直接 return）
4. **负数的运算要小心**：`-4 + (-2) = -6`，这是对的，表示 size = 6
