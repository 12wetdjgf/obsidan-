## 📚 第一步：先搞懂 LLRB 到底是什么

### 核心思想

你知道普通的 BST（二叉搜索树）吧？它有个致命缺点——如果你按顺序插入 1, 2, 3, 4, 5，它会变成一条"竹竿"（spindly tree），查找就退化成 O(N) 了，超慢的！

**2-3 树** 能自动保持平衡，但代码写起来——天哪——各种corner case能让人崩溃 😵

所以！天才们发明了 **LLRB（左倾红黑树）**：
> 它本质就是一棵 BST，但通过给节点"涂颜色"（红/黑），和 2-3 树形成 **一一对应** 的关系，从而"偷"到了 2-3 树的平衡性！

### ⚠️ 超重要：Link vs Node

这个实验室用的是 **给节点涂色**，不是给边（link）涂色！

关系是这样的：

```
讲座里的说法：           实验里的说法：
A --红色链接-- B    →    A 本身被涂成红色（A 是 B 的子节点）
```

**红色节点 = "我和我爸在 2-3 树里是同一个节点！"**

记住这个，后面全靠它！

## 🎨 第二步：LLRB 的四大铁律

这是你写代码的 **生命线**，务必刻进 DNA 里：

| # | 不变量（Invariant） | 为什么？ |
|---|---|---|
| 1 | 🖤 根节点必须是黑色 | 根没有父节点，不可能"属于"某个父节点的 2-3 节点 |
| 2 | ⬅️ 红色子节点必须在左边 | 这就是"左倾"的含义！ |
| 3 | 🚫 不能有两个红色子节点 | 否则对应的 2-3 节点有 3 个 key，爆了！ |
| 4 | 🚫 红色节点的父节点不能也是红色 | 同理，会让 2-3 节点有 3 个 key |

## 🔧 第三步：三大修复操作

插入新节点时，**永远先插入为红色节点**（就像在 2-3 树里先塞进已有节点）。但这可能打破铁律，所以我们需要三个"修复技能"：

### 技能 1：`rotateLeft(node)` —— 左旋

**触发条件**：右边出现了红色子节点（违反铁律 #2）

```
    a(黑)                b(黑)
     \        左旋        / 
      b(红)   →→→    a(红)
```

**关键操作**（用大白话说）：
1. `b` 升上去当新的根
2. `a` 变成 `b` 的左孩子
3. `b` 原来的左孩子 → 变成 `a` 的右孩子（不能丢！）
4. **颜色互换**：新根继承旧根的颜色，旧根变红

### 技能 2：`rotateRight(node)` —— 右旋

**触发条件**：出现连续两个左倾红色节点（违反铁律 #4）

```
      b(黑)              a(黑)
     /        右旋          \
    a(红)     →→→         b(红)
   /
  x(红)
```

**关键操作**（和左旋完全对称）：
1. `a` 升上去当新的根
2. `b` 变成 `a` 的右孩子
3. `a` 原来的右孩子 → 变成 `b` 的左孩子
4. **颜色互换**

### 技能 3：`flipColors(node)` —— 颜色翻转

**触发条件**：一个节点有两个红色子节点（违反铁律 #3）

```
      b(黑)              b(红)
     / \      翻转       / \
   a(红) c(红) →→→   a(黑) c(黑)
```

就是把自己和两个孩子的颜色全部取反！超简单的～

## 🧩 第四步：`insert` 的完整逻辑

这是最关键的部分！插入流程分两大块：

### Part A：标准 BST 插入
```
如果当前节点是 null → 创建新的红色节点，返回
如果 key < 当前节点的 key → 往左子树递归插入
如果 key > 当前节点的 key → 往右子树递归插入
```

### Part B：修复阶段（顺序很重要！！）

插入完之后，**从下往上**检查并修复，按这个顺序：

```
① 右孩子是红色 AND 左孩子不是红色？  → rotateLeft
② 左孩子是红色 AND 左孩子的左孩子也是红色？  → rotateRight  
③ 左孩子是红色 AND 右孩子也是红色？  → flipColors
```

用伪代码表示：

```
node = 正常BST插入(node, key)

if (isRed(node.right) && !isRed(node.left))     → node = rotateLeft(node)
if (isRed(node.left) && isRed(node.left.left))   → node = rotateRight(node)
if (isRed(node.left) && isRed(node.right))        → node = flipColors(node)

return node
```

> 🌟 **这三步的顺序不能乱！** 因为做了一步之后可能触发下一步。而且因为是递归的，修复会自动"向上传播"——就像 2-3 树里的 key 往上推一样！

最后别忘了：**在最外层的 `insert` 方法里，把根节点强制设为黑色！**

## 💡 实现提示（不给你代码，但给你钥匙🔑）

### `flipColors` 提示：
- 就三行代码：翻转 `node` 的颜色，翻转 `node.left` 的颜色，翻转 `node.right` 的颜色
- 翻转 = `!原来的颜色`（布尔取反）

### `rotateLeft(node)` 提示：
```java
// 1. 保存右孩子到临时变量 x
// 2. node 的右孩子 = x 的左孩子  （不能丢掉！）
// 3. x 的左孩子 = node            （node 降下去了）
// 4. x 的颜色 = node 的颜色       （继承旧根颜色）
// 5. node 的颜色 = RED             （降下去的变红）
// 6. return x                       （新的根）
```

### `rotateRight(node)` 提示：
- 和左旋完全对称！把 left/right 互换就行～

### `insert` 提示：
- 这是个递归方法
- 记住返回值是"修复后的新的子树根节点"
- 用 `isRed()` 方法来检查颜色，它已经帮你处理了 null 的情况（null 视为黑色）
