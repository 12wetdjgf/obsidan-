# 6.2 哈希冲突（Hash Collision）

两个不同的 key，哈希函数给出同一个下标，就叫**哈希冲突**。

```
"apple" → hash → 5
"mango" → hash → 5   ← 撞了！
```

## 解决方法一：链式地址法

每个格子放一条链表，冲突了就加到链表末尾。

```
下标 5: [apple,1] → [mango,2] → null
下标 6: [banana,3] → null
```

```java
class HashMapChaining {
    int capacity = 4;
    double loadThres = 0.75;  // 负载因子阈值
    LinkedList<int[]>[] buckets;

    HashMapChaining() {
        buckets = new LinkedList[capacity];
        for (int i = 0; i < capacity; i++)
            buckets[i] = new LinkedList<>();
    }

    int hash(int key) { return key % capacity; }

    String get(int key) {
        for (int[] pair : buckets[hash(key)])
            if (pair[0] == key) return String.valueOf(pair[1]);
        return null;
    }

    void put(int key, int val) {
        // 超过负载因子就扩容
        if ((double) size() / capacity > loadThres) extend();
        int idx = hash(key);
        for (int[] pair : buckets[idx]) {
            if (pair[0] == key) { pair[1] = val; return; }  // 更新
        }
        buckets[idx].add(new int[]{key, val});  // 新增
    }

    void extend() {
        // 容量翻倍，重新哈希所有元素
        LinkedList<int[]>[] old = buckets;
        capacity *= 2;
        buckets = new LinkedList[capacity];
        for (int i = 0; i < capacity; i++) buckets[i] = new LinkedList<>();
        for (LinkedList<int[]> bucket : old)
            for (int[] pair : bucket)
                put(pair[0], pair[1]);
    }

    int size() {
        int s = 0;
        for (LinkedList<int[]> b : buckets) s += b.size();
        return s;
    }
}
```

## 解决方法二：开放寻址法

不用链表，冲突了就在数组里**往后找空位**（线性探测）。

```
下标 5 被占了？试 6，6 也占了？试 7...
```

```java
class HashMapOpenAddr {
    int capacity = 4;
    int[] keys;
    int[] vals;
    int EMPTY = Integer.MIN_VALUE;
    int DELETED = Integer.MIN_VALUE + 1;  // 墓碑标记

    HashMapOpenAddr() {
        keys = new int[capacity];
        vals = new int[capacity];
        Arrays.fill(keys, EMPTY);
    }

    int hash(int key) { return (key % capacity + capacity) % capacity; }

    int get(int key) {
        int idx = hash(key);
        while (keys[idx] != EMPTY) {
            if (keys[idx] == key) return vals[idx];
            idx = (idx + 1) % capacity;  // 线性探测
        }
        return -1;  // 没找到
    }

    void put(int key, int val) {
        int idx = hash(key);
        while (keys[idx] != EMPTY && keys[idx] != DELETED) {
            if (keys[idx] == key) { vals[idx] = val; return; }
            idx = (idx + 1) % capacity;
        }
        keys[idx] = key;
        vals[idx] = val;
    }

    void remove(int key) {
        int idx = hash(key);
        while (keys[idx] != EMPTY) {
            if (keys[idx] == key) {
                keys[idx] = DELETED;  // 放墓碑，不能直接置空！
                return;
            }
            idx = (idx + 1) % capacity;
        }
    }
}
```

> 为什么删除要用墓碑？直接置空会断掉探测链，导致后续元素找不到！

## 负载因子

```
负载因子 = 元素数量 / 数组容量
```

- 负载因子 > 阈值（通常 0.75）→ 扩容，重新哈希
- 负载因子太低 → 浪费空间

## 两种方法对比

| | 链式地址 | 开放寻址 |
|--|---------|--------|
| 空间 | 额外链表开销 | 只用数组，紧凑 |
| 缓存 | 一般 | 友好（连续内存） |
| 删除 | 简单 | 需要墓碑标记 |
| 负载因子 | 可 > 1 | 必须 < 1 |

**口诀**：冲突两招：链式加链表，开放找空位。删除开放寻址要放墓碑！
