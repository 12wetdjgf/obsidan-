### Q1: 为什么计算机使用补码而不是原码？

**答**：

1. 统一加减运算（减法变加法）
2. 简化硬件电路设计
3. 只有一个零（原码有+0和-0）
4. 充分利用编码空间（多表示一个负数）

### Q2: 为什么 byte 的范围是 -128 ~ 127？

**答**：

- 8位补码：1000 0000 ~ 0111 1111
- 1000 0000 = -128（最小负数）
- 0111 1111 = 127（最大正数）
- 负数比正数多一个是因为0占了正数的一个位置

### Q3: 为什么浮点数不能精确表示 0.1？

**答**：

- 0.1 的二进制是无限循环小数：0.0001100110011...
- 计算机只能存储有限位，产生舍入误差
- 解决方法：使用误差范围比较，或使用 Decimal 类型

### Q4: UTF-8 和 Unicode 的区别？

**答**：

- Unicode 是字符集（规定了字符的编号）
- UTF-8 是编码方式（规定了如何存储）
- 类似于：Unicode 是门牌号，UTF-8 是地址格式

### Q5: 为什么选择 UTF-8 而不是 UTF-16？

**答**：

- UTF-8 兼容 ASCII（英文网站节省空间）
- UTF-8 变长编码更灵活
- UTF-8 没有字节序问题（大端/小端）
- 互联网标准，广泛支持

byte 1字节，short 2字节

### 6.2 进制转换技巧

**二进制 ↔ 十进制**：

```python
# 十进制 → 二进制
bin(13)  # '0b1101'

# 二进制 → 十进制
int('1101', 2)  # 13
```

**十六进制 ↔ 十进制**：

```python
# 十进制 → 十六进制
hex(255)  # '0xff'

# 十六进制 → 十进制
int('ff', 16)  # 255
```

### 6.3 字符编码实战

```python
# 字符 → Unicode 码点
ord('中')  # 20013 (十进制) = U+4E2D

# Unicode 码点 → 字符
chr(20013)  # '中'

# 字符串 → UTF-8 字节
'中'.encode('utf-8')  # b'\xe4\xb8\xad'

# UTF-8 字节 → 字符串
b'\xe4\xb8\xad'.decode('utf-8')  # '中'
```
