# 6.3 哈希算法（Hash Algorithm）

哈希算法把**任意长度的数据**映射成**固定长度哈希值**的函数。

## 好的哈希算法要满足

| 要求 | 说明 |
|------|------|
| 确定性 | 同样输入永远同样输出 |
| 高效率 | 计算要快 |
| 均匀分布 | 哈希值尽量散开，减少冲突 |
| 雪崩效应 | 输入微小变化 → 输出大幅变化 |
| 单向性 | 无法从哈希值反推输入（加密场景） |

## 常见哈希算法

| 算法 | 输出长度 | 用途 |
|------|---------|------|
| MD5 | 128 位 | 文件校验（已不安全） |
| SHA-256 | 256 位 | 密码学、区块链 |
| MurmurHash | 32/64 位 | 哈希表（性能优先） |

## 不同类型 key 的哈希计算

```java
// 整数：直接用值
int hash = key;

// 浮点数：转成二进制位
int hash = Float.floatToIntBits(key);

// 字符串：多项式哈希（最常用）
int hash = 0;
for (char c : str.toCharArray())
    hash = hash * 31 + c;  // 31 是经典魔法数字

// 对象：组合各字段
int hash = Objects.hash(field1, field2);
```

## Java 中的 hashCode()

所有 Java 对象都有 `hashCode()`，自定义类需要重写：

```java
class Point {
    int x, y;

    @Override
    public int hashCode() {
        return Objects.hash(x, y);  // 推荐写法
    }

    @Override
    public boolean equals(Object o) {
        if (!(o instanceof Point)) return false;
        Point p = (Point) o;
        return x == p.x && y == p.y;
    }
    // 规则：equals() 相等 → hashCode() 必须相同！
}
```

## 哈希表中的下标计算

```java
// 注意 Java 的 % 对负数可能返回负数！
// 安全写法：
int index = (key.hashCode() % capacity + capacity) % capacity;
```

## 为什么容量选质数？

容量选质数（11、13、17…）能让哈希值分布更均匀，减少冲突。
Java HashMap 用 2 的幂次 + 位运算 `hash & (n-1)` 替代取模，速度更快。

## 哈希 vs 加密

| | 哈希 | 加密 |
|--|------|------|
| 可逆？ | 不可逆 | 可逆（有密钥） |
| 用途 | 快速查找、校验 | 数据保密 |
| 例子 | HashMap、密码存储 | AES、RSA |

**口诀**：好哈希要均匀分布，字符串用多项式哈希，重写 hashCode 记得也重写 equals！
