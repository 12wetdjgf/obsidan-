## 🎓 核心概念速查

### Agent循环（s01）

```python
while True:
    response = client.messages.create(
        model=MODEL,
        system=SYSTEM,
        messages=messages,
        tools=TOOLS,
    )
    messages.append({"role": "assistant", "content": response.content})
    
    if response.stop_reason != "tool_use":
        return  # 完成
    
    # 执行工具
    results = [...]
    messages.append({"role": "user", "content": results})
```

**关键**：`stop_reason != "tool_use"` 是停止条件

### 工具分发（s02）

```python
TOOL_HANDLERS = {
    "bash": lambda **kw: run_bash(kw["command"]),
    "read_file": lambda **kw: run_read_file(kw["path"]),
    "write_file": lambda **kw: run_write_file(kw["path"], kw["content"]),

# 使用
handler = TOOL_HANDLERS.get(block.name)

```

**关键**：一个字典，一个查表，无限工具

### TodoManager（s03）

class TodoManager:
    def update(self, items: list) -> str:
        # 检查：只能有一个 in_progress

## 🔑 关键设计模式

### 模式1：工具分发

**问题**：如何优雅地添加工具？

**解决**：
```python
TOOL_HANDLERS = {name: handler}
handler = TOOL_HANDLERS[name]
```

**优点**：
- 添加工具只需一行
- 循环代码不变
- 易于扩展

---

### 模式2：上下文隔离

**问题**：长对话会污染上下文

**解决**：
```python
# 主Agent
messages = [...]

# 子Agent
sub_messages = []  # 干净！
```

**优点**：
- 主Agent保持清晰
- 子Agent独立工作
- 中间结果被丢弃

---

### 模式3：任务规划

**问题**：Agent容易忘记任务

**解决**：
```python
todo_manager = TodoManager()
if todo_manager.should_remind():
    inject_reminder()
```

**优点**：
- 完成率翻倍
- 自动提醒
- 任务可视化

## 💡 常见问题

### Q1: 为什么Agent会忘记任务？

**A**: 因为对话历史越来越长，系统提示被淹没了。

**解决**：
- 使用TodoList（s03）
- 使用Subagent（s04）
- 使用上下文压缩（s06）

---

### Q2: 如何添加新工具？

**A**: 4个步骤：

1. 定义schema
2. 实现处理函数
3. 添加到TOOLS
4. 添加到TOOL_HANDLERS

详见 `实战代码模板_下.md`

---

### Q3: 子Agent和主Agent有什么区别？

**A**:

| 特性 | 主Agent | 子Agent |
|------|---------|---------|
| messages | 保持 | 丢弃 |
| 工具 | 包含task | 不包含task |
| 返回值 | 文本 | 摘要 |
| 用途 | 管理 | 执行 |

---

### Q4: 为什么要用工具分发？

**A**: 因为：
- 易于添加工具
- 循环代码不变
- 代码更清晰
- 易于维护
